私も涼宮ハルヒに倣い、私の求める機能を満たすブログをとりあえずまず作ることにしました。

## なぜ今、自作ブログなのか

ご存知、ブログサービスは無料のものがいくつも提供されていますから、ブログ作成はあえて(意図して、強いて)行うことになります。今回はその理由を簡単に説明します。

### 既存のブログサービスへの不満

まずはじめに、**なぜどのブログサービスも白背景なのか、** という点を問い詰めたい。小 1 時間問い詰めたい。

真っ白な画面を直視すると眼精疲労で頭痛になり寝つきも悪くなるので、まとまった時間をかけて文章を読む場合に白背景は向かないと思っています。私はブラウザはSafari か Firefox を使っており、どちらも Web ページの背景色を書き換えて黒背景にする拡張機能または設定を使用していますので個人的に大きな問題はないのですが、やはり、はじめから黒背景を想定した Web デザインの方が好ましいと思います。また、押し付けがましいですが、今まで黒背景にあまり触れて来なかった人にとって私の作成したブログが黒背景に触れる機会になればいいなという野心を持っています。

既存の多くのブログサービスは執筆環境にも不満があります。Web 上のエディタどうしても動作が一定程度重くなりますし、ネットワーク接続が切れると作業が途中で止まってしまう可能性があります。記事のバックアップも各種ブログサービス内にあるとエクスポートが面倒です。Zenn の場合は[GitHub と連携してローカルのエディタで執筆することが可能](https://zenn.dev/zenn/articles/connect-to-github)で、これは非常に魅力的ですので、今回同じ手法で執筆、記事の管理ができるようにしています。

また、記事そのもの以外にも、編集履歴も残したいと考えています。基本的にどのブログサービスも投稿後の記事編集が可能ですが、投稿日と別に更新日が記載されたり、差分を確認できるサービスを提供しているのは Qiita だけのようです。これも魅力的な機能なので今回(簡易的ながら)採用しています。記事のフッターから GitHub 上の該当の.mdx ファイルを確認することができます。編集履歴はこちらからご参照ください。(なお、ブログ作成中のイけてない状態のコードを隠すために、ブログ完成以前のコミットは削除しています)

ここまで、機能的には Qiita と Zenn は光るところがあるのですが、これらは IT エンジニアが知識を共有する場を提供するサービスであり、記事のジャンルが限られる点が障害になります。私は IT 系の話に限らず、アニメや映画の感想、日記も投稿するつもりなので、前述のブログサービスは不採用になります。

以上から、上記不満を解消するため、下記要件を満たすブログを **「自分で作ればいい」** というところに落ち着きました。

- 背景色に黒を選択可能にする
- ローカルエディタで執筆
- 記事は Git でバージョン管理

### 技術的関心

私は IT エンジニアではありませんので、ふつーに生活しているとプログラミングに触れる機会はほぼありません。しかし、変動の激しい世の中では新しい技術には積極的に触れた方が良いと思われますし、縮小再生産を繰り返す人生を送りたくはありませんから、新しい趣味として今年からプログラミングに挑戦しています。その宿題として、ブログ作成を選定した側面もあります。Next.js、Docker(加えて、今回は最終的に不採用となりましたが、GitHub Actions)と最近のフロントエンド開発でもよく使用されていると思われる技術を纏めて触る良い機会にできました。

今後もデザインやホスティング先、デプロイ方法など変更したい要素はありますのでそれらを更新した際には記事にするかと思います。記事に纏め、第三者に伝わるよう表現することで自身も理解を深められる効果も期待しています。

私の技術的関心を満たすため、採用した技術は下記になります。

### インターネットの喧騒から距離を置く

「INTERNET YAMERO」とは言いませんが、TwitterやYoutubeやNoteといったファストなインターネットから距離を置く、ということもブログ作成の理由の一つになります。

時代を下る毎に SNS は「炎上」や「論争」が増えている(私のオススメ欄に多く入り込むようになった?)印象を受けます。しかしセンセーショナルな話題でアテンションを消費することは個人レベルでは百害あって一利無しですから、こんな状況の SNS などというものに触れる時間は少ない方が良いです。

少なくともこの記事を書いている時、私は VSCode とプレビュー用のブラウザしか見ていないので SNS から離れることができます。意味もなくついつい PC を触ってしまう時、取り敢えずここに立ち寄るようにすることで少なくとも SNS の接触時間を減らすことができます。触っていて components や layouts に気になる点が発生すれば、フロントエンドを勉強する良い機会になりますね。

こうしたインターネットの喧騒から距離を置き、静かに文章を読む環境を提供することを、評論家の宇野常寛氏が「遅いインターネット」運動と名付けて実践していますが[^1]、それの書く ver.というわけですね。
